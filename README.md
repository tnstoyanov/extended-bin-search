# BIN Search Web App

A modern, secure web application for searching and looking up Bank Identification Numbers (BINs) from a SQLite database.

## Features

- **Smart Search:** Enter a BIN and search by its exact length (6-11 digits)
- **Validation:** Red error message if input is outside 6-11 digit range
- **Secure Storage:** BIN data stored in encrypted SQLite database (not exposed via web)
- **Modern UI:** Responsive design with gradient styling
- **Fast Lookup:** Database-backed searches on server side
- **Clean Results:** Matches displayed in organized data table with small sans-serif fonts

## Installation

1. **Install dependencies:**
   ```bash
   npm install
   ```

2. **Prepare your data:**
   - Replace `data/bins_all.csv` with your BIN data
   - CSV must have headers: `BIN, Bank Name, Card Brand, Card Type, Country, Country Code`

3. **Initialize database:**
   ```bash
   npm run init-db
   ```
   This converts the CSV into a SQLite database (`bins.db`)

4. **Start the server:**
   ```bash
   npm start
   ```

5. **Open in browser:**
   - Navigate to `http://localhost:3000`

## Usage

1. Enter a BIN number (6-11 digits)
2. Click "Search"
3. Get results:
   - **Green message** with count if matches found
   - **Red message** "Your BIN should be 6-11 digits!" if input is invalid
   - **Red message** "No matches found!" if no results for valid input
   - Results appear in a formatted table below

## API

**POST** `/api/search`

Request:
```json
{
  "bin": "123456"
}
```

Response (on success):
```json
{
  "count": 3,
  "matches": [
    { "id": 1, "bin": "123456", "bank_name": "...", ... },
    ...
  ],
  "searchedBin": "123456",
  "binLength": 6
}
```

Response (on validation error):
```json
{
  "error": "Your BIN should be 6-11 digits!",
  "count": 0,
  "matches": []
}
```

## Security

- **Database Storage:** BIN data stored in SQLite database on server, not in memory or exposed to web
- **No CSV Exposure:** CSV is only used for initialization; data is in secure database
- **Server-Side Search:** All lookups happen server-side; no raw data sent to frontend
- **Single Button:** Simplified interface reduces attack surface

## Technology Stack

- **Backend:** Node.js, Express
- **Database:** SQLite3
- **Frontend:** HTML5, CSS3, Vanilla JavaScript
- **Data Import:** csv-parser

## Notes

- Database is initialized once via `npm run init-db`
- To reset database, run `npm run init-db` again
- For production, consider:
  - Adding authentication
  - Using environment variables for configuration
  - Implementing rate limiting
  - Adding HTTPS
  - Compressing database with encryption at rest
